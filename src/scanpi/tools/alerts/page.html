Topic format

scanpi/alerts/<severity>/<source>[/<suffix>]

severity = low | medium | high | critical
source   = gmrs | op25

Wildcard examples:
  scanpi/alerts/#                # everything
  scanpi/alerts/critical/#       # only critical
  scanpi/alerts/+/op25           # all P25 alerts

Payload is JSON: {ts, source, channel_or_tg, transcript, matched_rules, audio_url, severity, hits[]}

ntfy.sh — easiest free push (Android + iOS app)

Run once on your LAN. Subscribers point their app at http://your-pi:8080/scanpi-alerts.

# /etc/systemd/system/ntfy-bridge.service
[Unit]
Description=MQTT -> ntfy alert bridge
After=network-online.target

[Service]
ExecStart=/usr/bin/python3 /opt/ntfy_bridge.py
Restart=always
User=scanpi

[Install]
WantedBy=multi-user.target
# /opt/ntfy_bridge.py — about 20 lines
import json, paho.mqtt.client as mqtt, urllib.request

NTFY = "http://localhost:8080/scanpi-alerts"

def on_msg(c, u, m):
    p = json.loads(m.payload.decode())
    title = f"[{p['severity'].upper()}] {p.get('channel_or_tg','')}".strip()
    body  = p.get("transcript", "")
    headers = {"Title": title, "Priority":
               {"critical":"5","high":"4","medium":"3","low":"2"}.get(p["severity"],"3"),
               "Tags": ",".join(p.get("matched_rules", []))}
    urllib.request.urlopen(urllib.request.Request(NTFY, data=body.encode(), headers=headers))

c = mqtt.Client()
c.connect("localhost", 1883, 60)
c.subscribe("scanpi/alerts/#"); c.on_message = on_msg
c.loop_forever()

Pushover — for paid push w/ priority + sounds

Drop-in script that subscribes to MQTT and forwards to the Pushover API.

# pushover_bridge.py — set PUSHOVER_USER + PUSHOVER_TOKEN env vars
import json, os, urllib.request, urllib.parse, paho.mqtt.client as mqtt

USER, TOKEN = os.environ["PUSHOVER_USER"], os.environ["PUSHOVER_TOKEN"]
PRIORITY = {"low":-1, "medium":0, "high":1, "critical":2}

def on_msg(c, u, m):
    p = json.loads(m.payload.decode())
    data = urllib.parse.urlencode({
        "user": USER, "token": TOKEN,
        "title": f"[{p['severity'].upper()}] {p.get('channel_or_tg','')}",
        "message": p.get("transcript", ""),
        "priority": PRIORITY.get(p["severity"], 0),
        # priority=2 needs retry+expire:
        "retry": 60, "expire": 600,
        "sound": "siren" if p["severity"] == "critical" else "pushover",
    }).encode()
    urllib.request.urlopen("https://api.pushover.net/1/messages.json", data=data)

c = mqtt.Client(); c.connect("localhost", 1883, 60)
c.subscribe("scanpi/alerts/#"); c.on_message = on_msg
c.loop_forever()

Home Assistant — MQTT integration

Add this to configuration.yaml; alerts show up as a sensor + trigger automations.

mqtt:
  sensor:
    - name: "ScanPi Alert"
      state_topic: "scanpi/alerts/+/+/#"
      value_template: "{{ value_json.severity }}"
      json_attributes_topic: "scanpi/alerts/+/+/#"

automation:
  - alias: "ScanPi critical alert"
    trigger:
      - platform: mqtt
        topic: "scanpi/alerts/critical/#"
    action:
      - service: notify.mobile_app_my_phone
        data:
          title: "ScanPi CRITICAL"
          message: "{{ trigger.payload_json.transcript }}"
          data:
            channel: alerts
            ttl: 0
            priority: high

Tasker (Android) — MQTT plugin

Use the "MQTT Client" plugin from Play Store. Configure:

Broker:    your-pi.local:1883
Topic:     scanpi/alerts/#
QoS:       1
On message Tasker task:
  1. Variable Set %sev TO %mqtt_payload (parse JSON: $.severity)
  2. Notify  Title=ScanPi %sev  Text=%mqtt_payload (parse $.transcript)
  3. If %sev == critical -> Vibrate Pattern: 0,400,200,400

See tools/alerts/SUBSCRIBERS.md on the Pi for the full deployment guide.
